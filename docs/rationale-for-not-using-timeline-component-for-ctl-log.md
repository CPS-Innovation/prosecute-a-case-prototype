Just wanted to share some thoughts on using the timeline component for the CTL log.

Full disclosure: I designed/contributed the timeline component that's in the MoJ design system (and HMCTS design system before that).

We used it for a while in the timeline and activity log within the ‘Manage teacher training applications’ case working system at DfE.

But we ended up removing it as noted in this design history:

https://becoming-a-teacher.design-history.education.gov.uk/manage-teacher-training-applications/improving-the-activity-log-and-timeline/

“We also moved away from the blue ‘train track’ styling which did not work very well, especially on the activity log where it implied a connection between events in different applications.”

And:

“The new styling will also work better than the ‘train track’ when we introduce the filter panel on the left hand side of the page.”

It’s okay when all you need to do is show what happened and when but when you need to show more detail or take action it starts to break down quickly. For example, if you have search and filter, which could easily be needed for CTL log, then the timeline view could be confusing and misleading because it will show one event immediately preceding another when there are others in between.

So we moved away from that and you’ll see that’s the style we’ve been using throughout my prototype for activity log. It’s also just less fussy and more consistent with all other types of lists.

Just thought I’d flag.
